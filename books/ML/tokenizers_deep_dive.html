Tokenizers: Text as Model Input

Tokenization is the contract between human text and model tensors. This page covers normalization, pre-tokenization, BPE-style merges, WordPiece scoring, unigram tokenization, special tokens, span alignment, and context budgeting.

1. Tokenizer Contracts

A tokenizer does not merely split strings. It defines how text becomes ids, how offsets map back to original characters, which special tokens mark boundaries, and how many model positions are consumed.

Pipeline

raw text
-> normalization       # Unicode, case, whitespace, accents
-> pre-tokenization    # words, punctuation, bytes, regex chunks
-> model tokenization  # BPE, WordPiece, Unigram, byte-level rules
-> post-processing     # BOS/EOS/SEP, chat template, truncation
-> tensors             # input_ids, attention_mask, token_type_ids

Why boundaries matter

Token boundaries are not word boundaries. A single word can become many tokens, and a single token can include a leading space. This affects cost, context windows, character span labels, retrieval chunking, and streaming output.

For NER and extraction, always keep offset mappings. For LLM prompts, count tokens after applying the actual chat or instruction template.

2. Tokenizer Families

FamilyMechanicsCommon detail
BPEStarts from small symbols and repeatedly merges frequent adjacent pairs.Fast, common in GPT-style tokenizers; byte-level BPE can represent any byte sequence.
WordPieceChooses subwords with a likelihood-like score, often using continuation markers such as ##ing.Common in BERT-style encoders.
UnigramStarts with a large candidate vocabulary and prunes pieces based on a probabilistic model.Can choose alternate segmentations; common in SentencePiece workflows.
Character/byteEvery character or byte is a token.Robust coverage, but longer sequences and harder long-range modeling.

3. Edge Cases

Spaces and punctuation

Many tokenizers encode leading spaces into tokens, so "cat" and " cat" can be different ids. Punctuation often becomes separate tokens unless frequent pairs have been merged.

Code, math, and rare names

Identifiers such as HTTPRequestHandler, paths, formulas, and mixed-language names can explode into many tokens. That increases latency and makes span alignment harder.
