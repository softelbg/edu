Deep learning code becomes readable when each module is treated as a contract: it accepts a tensor of a known rank, changes a predictable set of dimensions, and preserves the rest.

1. Tensor Contracts

A tensor is not just a multidimensional array. In a model it carries semantics: batch, channel, time, height, width, embedding dimension, head, class, sequence position. Most model bugs are contract bugs: the values may be legal floating point numbers, but the axes no longer mean what the next layer expects.

Domain Common shape Meaning Typical layers
Tabular [N, F] N examples, F features. Linear, BatchNorm1d, activations, dropout.
Images [N, C, H, W] Batch, channels, height, width. PyTorch is channel-first by default. Conv2d, BatchNorm2d, pooling, AdaptiveAvgPool2d.
Audio / signals [N, C, L] Batch, channels, sequence length. Conv1d, MaxPool1d, LayerNorm, RNNs.
Volumes [N, C, D, H, W] Batch, channels, depth, height, width. Conv3d, InstanceNorm3d, MaxPool3d.
Text tokens [N, T] then [N, T, E] Token ids become embedding vectors. Embedding, MultiheadAttention, TransformerEncoderLayer.
Rule of thumb: write the shape above every block before writing code. If a dimension changes, name the operator responsible for changing it. If no operator owns the change, you have a hidden bug.

The module view

torch.nn.Module gives layers two essential responsibilities: store learnable parameters and define a forward mapping. Some modules have weights, such as Linear, Conv2d, Embedding, and BatchNorm2d. Others are deterministic transforms, such as Flatten, ReLU, MaxPool2d, and Softmax. Both types are layers because both shape the computation graph.

class SmallCNN(nn.Module):
    def __init__(self, classes=10):
        super().__init__()
        self.features = nn.Sequential(
            nn.Conv2d(3, 32, kernel_size=3, padding=1),   # [N, 3, H, W] -> [N, 32, H, W]
            nn.BatchNorm2d(32),
            nn.ReLU(inplace=True),
            nn.MaxPool2d(2),                              # [N, 32, H/2, W/2]
            nn.Conv2d(32, 64, kernel_size=3, padding=1),
            nn.ReLU(inplace=True),
            nn.AdaptiveAvgPool2d((1, 1)),                 # [N, 64, 1, 1]
        )
        self.classifier = nn.Linear(64, classes)

    def forward(self, x):
        x = self.features(x)
        x = torch.flatten(x, 1)
        return self.classifier(x)

2. CNN Mechanics

A convolutional neural network learns local filters. A filter slides across spatial positions, takes dot products with local input patches, and produces feature maps. Stacking these maps lets the network build edges, textures, parts, and object-level evidence.

Convolution as local weight sharing

In a dense layer, every output unit receives every input feature with its own weight. In a convolution, the same small kernel is reused at every location. This makes the model translation-aware and parameter-efficient.

out[n, c_out, y, x] = bias[c_out] + sum over c_in, ky, kx: weight[c_out, c_in, ky, kx] * input[n, c_in, y * stride_y + ky * dilation_y - pad_y, x * stride_x + kx * dilation_x - pad_x]

The output spatial size for one dimension is:

L_out = floor((L_in + 2*padding - dilation*(kernel_size - 1) - 1) / stride + 1)

What the hyperparameters do

  • kernel_size controls the local receptive field.
  • stride skips positions and usually reduces spatial resolution.
  • padding adds boundary values so edge pixels can participate.
  • dilation spaces kernel taps apart, increasing receptive field without more parameters.
  • groups splits channels. groups=in_channels creates depthwise convolution.
  • bias adds one scalar per output channel unless disabled.

Channels, filters, and feature maps

A color image with shape [N, 3, H, W] has three input channels. A Conv2d(3, 32, 3) layer has 32 output filters. Each filter owns a [3, 3, 3] block of weights, one 3 by 3 kernel per input channel. The layer sums across the channel dimension and produces 32 feature maps.

Convolution type Meaning Parameter count without bias
Standard Every output channel sees every input channel. out_channels * in_channels * kH * kW
Grouped Input and output channels are split into independent groups. out_channels * (in_channels / groups) * kH * kW
Depthwise One spatial filter per input channel, usually followed by a 1x1 pointwise convolution. in_channels * depth_multiplier * kH * kW
Transposed Learns an upsampling-like mapping by spreading input values across a larger output grid. in_channels * out_channels/groups * kH * kW

4. Torch Layer Atlas

This atlas covers the main torch.nn layer families used in modern models. The examples focus on contracts and when to reach for each layer.

0 layers

Layer family notes

Containers and shape tools

Sequential is appropriate when each module has exactly one input and one output. ModuleList and ModuleDict register submodules without defining a forward path, which is useful when the forward pass branches, loops, or chooses layers dynamically. Flatten, Unflatten, Identity, LazyLinear, and lazy convolution layers are contract-management helpers.

Dense layers

Linear acts on the last dimension. If the input is [N, T, E], Linear(E, M) returns [N, T, M]. This behavior is why transformers use linear projections for queries, keys, values, feed-forward blocks, and classifiers without flattening the sequence dimension.

Convolutions

Conv1d, Conv2d, and Conv3d are the same idea applied to different spatial ranks. ConvTranspose* layers are not inverse convolutions in the algebraic sense, but they correspond to the transpose of the matrix that would implement convolution. They are common in decoders, segmentation heads, and generative models.

Pooling

Fixed pooling uses a local window. Adaptive pooling chooses the kernel and stride needed to hit a target output size. That makes AdaptiveAvgPool2d((1, 1)) a common bridge from variable-size feature maps to a fixed-size classifier.

Normalization

BatchNorm uses batch statistics during training and running estimates during evaluation. LayerNorm normalizes within each sample and is stable for sequence models and small batches. GroupNorm is a strong CNN choice when batch size is too small for reliable batch statistics. InstanceNorm normalizes each sample and channel independently, often in style transfer and image generation.

Activations

ReLU is cheap and sparse. GELU and SiLU are smooth and common in transformers and efficient CNNs. Sigmoid and Tanh saturate, so they are more often used for gates or bounded outputs than as default hidden activations.

Embeddings, recurrent layers, and attention

Embedding turns integer ids into vectors. RNN, GRU, and LSTM scan sequences with hidden state. MultiheadAttention lets every position read from every other position through learned similarity, and TransformerEncoderLayer packages attention, feed-forward layers, normalization, residual paths, and dropout.

5. Building CNN Architectures

A CNN architecture is a sequence of resolution changes and channel changes. Good designs state both clearly.

Canonical blocks

Block Typical code Why it works
Conv-BN-ReLU Conv2d -> BatchNorm2d -> ReLU Convolution extracts local features, normalization stabilizes scale, activation adds nonlinearity.
Downsample stride=2 conv or MaxPool2d(2) Reduces spatial cost while increasing receptive field.
Residual y = block(x) + shortcut(x) Preserves an easy gradient path and lets blocks learn refinements.
Depthwise separable groups=in_channels then 1x1 Conv2d Separates spatial filtering from channel mixing, reducing parameters and compute.
Global pooling head AdaptiveAvgPool2d(1) -> Flatten -> Linear Accepts flexible image sizes and avoids a large fully connected tail.

Common CNN failure: flattening too early. A fully connected layer after a large feature map can dominate parameter count. Prefer several convolutional blocks followed by global pooling unless the task truly needs location-specific dense weights.

6. Training Stack

Layers define the forward computation. Training adds data, loss, optimizer, precision policy, initialization, regularization, and evaluation mode.

Loss modules

Losses in torch.nn are modules too. They consume predictions and targets, then return a scalar or per-example tensor.

  • CrossEntropyLoss: multiclass logits plus integer class targets.
  • BCEWithLogitsLoss: binary or multilabel logits.
  • MSELoss: regression with squared error.
  • L1Loss and SmoothL1Loss: robust regression.
  • NLLLoss: log probabilities, often after LogSoftmax.
  • KLDivLoss: distribution matching.
  • TripletMarginLoss: metric learning.
  • CTCLoss: unaligned sequence labeling.

Training and eval mode

model.train() enables training behavior for layers such as dropout and batch normalization. model.eval() disables dropout and makes batch normalization use running estimates. It does not disable gradients; use torch.no_grad() or torch.inference_mode() for evaluation without gradient tracking.

model.train()
for x, y in loader:
    optimizer.zero_grad(set_to_none=True)
    logits = model(x)
    loss = criterion(logits, y)
    loss.backward()
    optimizer.step()

Initialization

Initialization controls signal scale before learning starts. ReLU-like networks usually pair well with Kaiming initialization. Tanh-like networks often use Xavier initialization. PyTorch layers have sensible defaults, but explicit initialization is useful when building unusual residual, attention, or normalization-free networks.

Regularization

Regularization limits overfitting. CNNs often combine data augmentation, weight decay, dropout in classifier heads, label smoothing, early stopping, and stochastic depth in deep residual families. Normalization layers are not regularizers by definition, but their statistical effects often improve optimization and generalization.

7. Debugging Deep Models

Debugging is faster when you inspect contracts, values, and training state separately.

Symptom Likely cause Check
Shape mismatch at Linear Flattened feature count changed after convolution or pooling. Print x.shape before Flatten, or use AdaptiveAvgPool2d.
Loss is nan Too high learning rate, unstable logits, bad normalization, invalid targets. Check input scale, target range, gradient norm, mixed precision scaler.
Accuracy stuck at chance Labels misaligned, wrong loss, final activation mismatch, model in eval mode. Overfit one small batch. It should reach near-zero loss.
Train good, validation poor Overfitting or data split mismatch. Add augmentation, weight decay, simpler head, inspect split leakage.
BatchNorm unstable Small batches or distribution shift. Try GroupNorm, freeze BatchNorm, or accumulate larger batches.

Forward hooks for inspection

def hook(name):
    def fn(module, inputs, output):
        x = output if torch.is_tensor(output) else output[0]
        print(name, tuple(x.shape), x.detach().mean().item(), x.detach().std().item())
    return fn

for name, module in model.named_modules():
    if isinstance(module, (nn.Conv2d, nn.Linear, nn.BatchNorm2d)):
        module.register_forward_hook(hook(name))
