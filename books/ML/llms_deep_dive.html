LLMs as Probabilistic Text Systems

An LLM predicts token distributions. Useful assistants emerge when that core model is wrapped with prompt contracts, decoding controls, retrieval, tools, memory, safety gates, and evaluation loops.

1. Model Contract

A decoder LLM consumes token ids and returns logits for the next token. Everything else is system design: templates, instructions, retrieval, tool schemas, output parsers, memory, and evaluation.

Autoregressive loop

tokens = tokenize(prompt)
while not done:
    logits = model(tokens, kv_cache)
    next_token = decode(logits, temperature, top_p, penalties)
    tokens.append(next_token)

Pretraining vs adaptation

Pretraining teaches broad next-token prediction. Instruction tuning teaches response style and task following. Preference optimization or reinforcement-style alignment adjusts behavior toward selected outputs. Retrieval and tools add external information and actions without changing model weights.

2. Decoding Controls

ControlEffectFailure mode
TemperatureScales logits before softmax. Lower is sharper, higher is more exploratory.Too low can be repetitive; too high can be incoherent.
Top-kKeeps only the most likely k candidates.Can remove rare but correct tokens.
Top-pKeeps the smallest set whose probability mass reaches p.Candidate count changes by confidence, which can surprise tests.
Repetition penaltyReduces probability of already used tokens.Can hurt exact code, names, or citations.
Stop sequencesTerminate generation when a delimiter appears.Wrong stop strings truncate valid content.

3. RAG, Tools, and Structured Outputs

RAG pattern

Retrieval-augmented generation usually means: chunk documents, embed chunks, retrieve candidates, optionally rerank, pack evidence into the prompt, generate an answer, and evaluate whether the answer is supported by retrieved evidence.

Tool-calling pattern

A tool-capable LLM chooses a structured action from a schema. The application validates arguments, executes the tool, adds tool results to the context, and asks the model to continue. Never treat tool arguments as trusted just because they were model-generated.
