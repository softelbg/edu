Computer vision models are shape-sensitive systems. The input may be an image, but the model sees a tensor with channels, height, width, dtype, normalization, and a label contract.

1. Image Contracts

Representation Typical shape Meaning Common pitfalls
Single image [C, H, W] or [H, W, C] Channels, height, width. PyTorch commonly uses channel-first. Mixing RGB/BGR order, forgetting normalization, inconsistent resize.
Batch [N, C, H, W] Mini-batch of images. Variable sizes require padding, cropping, or special collate logic.
Classification target [N] or one-hot [N, K] One class id or a multi-label vector. Using softmax for multi-label instead of sigmoid.
Detection target boxes [M, 4], labels [M] Object rectangles, usually x1, y1, x2, y2. Wrong coordinate format after resize, crop, or flip.
Segmentation target [H, W], [K, H, W], or instance masks Pixel-level class or instance assignment. Resizing masks with bilinear interpolation instead of nearest-neighbor.
Rule of thumb: every transform must update every annotation it touches. A crop changes images, boxes, masks, and keypoints. A brightness change usually changes only pixels. A horizontal flip changes pixels and geometry, but not class labels.

Preprocessing contract

Most vision models expect a fixed value range and normalization. A common pipeline is decode image, convert channel order, resize or crop, convert to floating point, scale to [0, 1], then normalize with dataset mean and standard deviation.

# Typical PyTorch-style image preprocessing contract
image_uint8:          [H, W, 3], RGB, values 0..255
to_tensor:            [3, H, W], float32, values 0..1
normalize(mean, std): [3, H, W], approximately centered channels
batch:                [N, 3, H, W]

2. Filters and Features

Before deep networks, vision pipelines relied heavily on handcrafted filters. Modern CNNs learn filters, but the same mechanics still explain edges, textures, receptive fields, and feature maps.

Filtering as convolution

A filter replaces each pixel with a weighted sum of a local neighborhood. Different kernels emphasize different structures: blur averages noise, sharpen boosts high frequencies, Sobel filters approximate gradients, and Laplacian filters highlight rapid changes.

output[y, x] = sum over ky, kx: kernel[ky, kx] * image[y + ky - pad, x + kx - pad]

From filters to features

  • Edges: strong local intensity change.
  • Corners: intensity changes in multiple directions.
  • Texture: repeated local patterns.
  • Keypoints: stable local landmarks.
  • Feature maps: spatial responses to learned or fixed filters.

3. Core Computer Vision Tasks

Different tasks answer different questions about an image. The model output shape and the metric must match the question.

Task Question Prediction Typical models
Image classification What is in the image? Class logits or probabilities. ResNet, EfficientNet, ViT, ConvNeXt, linear probes.
Multi-label classification Which labels are present? Independent sigmoid scores per label. CNN/ViT backbones with binary heads.
Object detection What objects are present and where? Boxes, class scores, confidence scores. Faster R-CNN, RetinaNet, YOLO, DETR, SSD.
Semantic segmentation What class is each pixel? Class map or per-class logits per pixel. FCN, U-Net, DeepLab, SegFormer.
Instance segmentation Which pixels belong to each object instance? Boxes, labels, scores, and instance masks. Mask R-CNN, YOLACT, SOLO, SAM-style mask heads.
Keypoint detection Where are structured landmarks? Point coordinates or heatmaps. HRNet, OpenPose-style heads, heatmap regressors.
Depth and flow How far or how does motion change over pixels? Dense scalar or vector fields. Encoder-decoder models, stereo networks, optical-flow networks.

4. Evaluation Metrics

Vision metrics are not interchangeable. A detector with high classification accuracy may localize poorly. A segmenter with high pixel accuracy may miss rare classes.

Classification metrics

  • Accuracy: fraction of images with correct top-1 prediction.
  • Top-k accuracy: target appears in the k highest scores.
  • Precision: predicted positives that are correct.
  • Recall: real positives that were found.
  • F1: harmonic mean of precision and recall.
  • Calibration: predicted confidence matches empirical correctness.

Detection and mask metrics

IoU = area(prediction intersect ground_truth) / area(prediction union ground_truth) Dice = 2 * intersection / (prediction_area + ground_truth_area) precision = TP / (TP + FP) recall = TP / (TP + FN)

mAP averages precision-recall quality across classes and IoU thresholds. Segmentation often reports mean IoU, Dice, pixel accuracy, boundary F-score, and per-class recall.

Metric trap: pixel accuracy can look excellent when the background dominates. For medical, industrial, satellite, and safety tasks, report per-class IoU, Dice, recall, and failure examples, not only one aggregate number.

7. CV Workflow

Vision systems are data systems. Annotation definitions, split design, image quality, deployment constraints, and review tooling matter as much as architecture.

Stage Checklist Failure to avoid
Data definition Define classes, box policy, mask boundaries, ignore regions, occlusion rules. Annotators label the same visual case differently.
Splitting Split by scene, patient, camera, video, user, or time when needed. Near-duplicate frames leak between train and validation.
Training Use augmentations that preserve labels; monitor per-class metrics. Augmentation silently breaks boxes or masks.
Thresholding Choose score thresholds for operational precision/recall targets. Reporting only default threshold performance.
Deployment Measure latency, memory, input resolution, camera differences, drift. High offline accuracy but unacceptable runtime behavior.

Evaluation report minimum

  • Dataset version, split definition, image count, object count, class distribution.
  • Preprocessing and input resolution.
  • Metric definitions, IoU thresholds, confidence thresholds, and ignored labels.
  • Aggregate score plus per-class score.
  • Confusion matrix for classification or detection class confusions.
  • Qualitative panels: true positives, false positives, false negatives, boundary failures.

8. Debugging CV Models

Most CV bugs are visible if you draw the data after every transform. Do not debug box or mask pipelines from arrays alone.

Symptom Likely cause Check
Good training score, poor validation Leakage, overfit augmentations, split mismatch, label noise. Inspect near-duplicates and validate by scene/entity/time.
Boxes are consistently shifted Resize/crop coordinate bug or wrong box format. Draw boxes before and after every transform.
Masks have jagged or mixed labels Mask resized with bilinear interpolation. Use nearest-neighbor for class masks.
High pixel accuracy, low IoU Background dominates the image. Report per-class IoU and foreground Dice.
Detector has many duplicate boxes NMS threshold too high or model emits overlapping anchors. Sweep confidence and NMS IoU; plot precision-recall.
