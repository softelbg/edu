1. Attention Mechanics

For each head, a token representation is projected into query, key, and value vectors. Query-key dot products produce attention logits. Softmax turns logits into weights. Weighted values produce the next representation.

Scaled dot-product attention

Q = X Wq
K = X Wk
V = X Wv
scores = Q K^T / sqrt(d_head)
scores = scores + mask
weights = softmax(scores)
output = weights V

Why scale by sqrt(d)

Dot products grow in magnitude with vector dimension. Without scaling, softmax saturates too early, attention becomes nearly one-hot, and gradients through non-winning keys shrink. Scaling keeps logits in a trainable range.

3. Masks, Positions, and Cache

Mask types

Padding masks prevent tokens from reading padded positions. Causal masks prevent a decoder token from reading future tokens. Cross-attention masks control how decoder positions read encoder positions.

KV cache

During autoregressive generation, previous keys and values are reused. The model computes Q for the new token and attends over cached K/V from prior tokens, avoiding a full recomputation of the prefix each step.
