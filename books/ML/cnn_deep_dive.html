1. Shape Math

A 2D convolution layer maps [N, C_in, H, W] to [N, C_out, H_out, W_out]. The batch dimension is preserved. The channel count becomes the number of learned filters. Height and width depend on kernel, stride, padding, and dilation.

Output size

For each spatial axis:

effective_kernel = dilation * (kernel - 1) + 1
out = floor((input + 2 * padding - effective_kernel) / stride) + 1

If this value is not positive, the layer cannot be applied. If it is not the size you expected, inspect padding and stride before changing model code.

Parameter count

For ordinary convolution:

weights = out_channels * (in_channels / groups) * kernel_h * kernel_w
bias = out_channels if bias=True else 0

Depthwise convolution is the special case groups = in_channels. It saves parameters by filtering each channel separately, usually followed by a 1x1 pointwise convolution to mix channels.

A CNN layer is translation equivariant, not fully translation invariant. If an object shifts, the feature map shifts. Pooling, striding, data augmentation, and classifier heads create practical tolerance to shifts.

2. Filters, Channels, and Nonlinearity

What a filter learns

A filter is a small tensor that scores a local patch. Early filters often behave like edge, color, and texture detectors. Deeper filters respond to parts, motifs, or task-specific patterns because their receptive fields cover larger regions of the original image.

Each output channel is one learned feature detector applied across the whole image. Sharing the same weights at every location is what makes convolution efficient.

Why activation matters

Stacking linear convolutions without nonlinear activations collapses into another linear operation. ReLU, GELU, or SiLU lets the network compose piecewise nonlinear decisions: edge to corner, corner to part, part to object.

x = conv(x)
x = norm(x)       # optional but common
x = activation(x)
x = pooling(x)    # optional resolution change

3. Architecture Design

Design choiceWhat it controlsTypical failure
Kernel sizeLocal pattern width. 3x3 stacks are common because they add nonlinearity while growing receptive field gradually.Very large kernels early can waste parameters and blur local detail.
Stride and poolingSpatial downsampling and compute reduction.Downsampling too early destroys small objects and thin structures.
ChannelsNumber of feature detectors per resolution.Too few channels underfit; too many make dense heads expensive.
Residual pathsGradient flow and feature reuse.Shape mismatch when channel or stride changes without projection.
Global average poolingConverts feature maps to class features without huge flatten layers.Loses location-specific evidence when exact geometry is needed.
 
class SmallResidualCNN(nn.Module):
    def __init__(self, num_classes):
        super().__init__()
        self.stem = nn.Sequential(
            nn.Conv2d(3, 32, kernel_size=3, padding=1, bias=False),
            nn.BatchNorm2d(32),
            nn.SiLU()
        )
        self.block = nn.Sequential(
            nn.Conv2d(32, 32, 3, padding=1, bias=False),
            nn.BatchNorm2d(32),
            nn.SiLU(),
            nn.Conv2d(32, 32, 3, padding=1, bias=False),
            nn.BatchNorm2d(32)
        )
        self.head = nn.Sequential(nn.AdaptiveAvgPool2d(1), nn.Flatten(), nn.Linear(32, num_classes))

    def forward(self, x):
        x = self.stem(x)
        x = F.silu(x + self.block(x))
        return self.head(x)
