Classic machine learning is a collection of strong inductive biases. Each algorithm makes a bet about what structure matters: distance, margin, density, linearity, probability, axis-aligned rules, or low-dimensional variance.

1. Algorithm Map

Family Core idea Best when Watch out for
Linear models Prediction is a weighted sum of features. Signals are additive, data is high-dimensional, interpretability matters. Underfitting nonlinear patterns unless features are engineered.
Kernel and margin methods Choose a separating surface with a large margin. Medium-size datasets with clean geometry and strong class separation. Scaling to huge datasets and tuning kernels.
Neighbor methods Predict from nearby labeled examples. Local similarity is meaningful and feature scaling is controlled. Slow inference and high-dimensional distance collapse.
Trees and ensembles Partition feature space with rules, then average or vote many learners. Tabular data, nonlinear interactions, mixed feature types. Extrapolation, leakage, and careless validation.
Clustering Find unlabeled groups by distance, density, or probability. Exploration, segmentation, compression, anomaly discovery. Clusters may be artifacts of scale or assumptions.
Dimensionality reduction Represent data with fewer coordinates while preserving structure. Visualization, compression, denoising, feature preprocessing. Losing supervised signal while preserving unsupervised variance.
Feature matrix contract: most classic ML estimators consume X with shape [n_samples, n_features]. Supervised estimators also consume y, usually [n_samples]. Every transformation should preserve sample order.

2. Supervised Learning

Supervised algorithms learn a mapping from features to labels or targets. Classification predicts categories; regression predicts numeric values.

Regression

Linear regression chooses coefficients that minimize squared error. Ridge regression adds an L2 penalty that shrinks coefficients. Lasso adds an L1 penalty that can set some coefficients to exactly zero.

linear: y_hat = w0 + w1*x1 + ... + wp*xp least squares: min_w sum_i (y_i - y_hat_i)^2 ridge: min_w loss + alpha * ||w||_2^2 lasso: min_w loss + alpha * ||w||_1

Classification

Logistic regression turns a linear score into a probability. SVMs use hinge loss and focus on margin violations. Naive Bayes models class-conditional feature likelihoods. kNN delays learning until prediction time and votes among neighbors.

logistic: P(y=1|x) = sigmoid(w dot x + b) hinge: max(0, 1 - y * score) kNN: vote(labels of k nearest training samples)

Scaling is part of the model

Distance-based, margin-based, and regularized linear models are sensitive to feature scale. A feature measured in thousands can dominate a feature measured in fractions. Tree models are much less sensitive to monotonic scaling because they split by thresholds.

3. Unsupervised Learning

Unsupervised learning has no target column. It asks the model to expose structure already present in X.

Algorithm Assumption Output Main knobs
K-means Clusters are compact and roughly spherical. Centroids and cluster assignments. k, initialization, distance scaling.
Gaussian mixture Data is a mixture of Gaussian components. Soft cluster probabilities and component parameters. Number of components, covariance type, regularization.
DBSCAN Clusters are dense regions separated by sparse regions. Cluster labels plus noise points. eps, min_samples.
Hierarchical clustering Nested group structure is meaningful. Dendrogram or flat clustering at a chosen cut. Linkage and distance metric.
PCA High-variance linear directions capture useful structure. Principal axes, projections, explained variance. Number of components, scaling, whitening.
Unsupervised warning: clusters are not automatically real-world categories. They are model outputs under a distance, scale, and objective. Validate them with domain knowledge or downstream usefulness.

4. Trees and Ensembles

Decision trees split feature space into rectangles. Random forests reduce variance by averaging many noisy trees. Boosting reduces bias by adding small trees that fix previous errors.

Decision tree split

A classification tree searches for a feature and threshold that create purer child nodes. Gini impurity is common:

Gini(node) = 1 - sum_c p(c)^2 gain = impurity(parent) - weighted_impurity(children)

A single deep tree can memorize. Pruning, minimum leaf sizes, and maximum depth reduce variance.

Ensemble logic

  • Bagging: train models on bootstrapped datasets and average predictions.
  • Random forest: bagged trees plus random feature subsets at each split.
  • Extra trees: use more random split thresholds to reduce correlation.
  • Gradient boosting: add trees sequentially to correct residuals or gradients.
  • AdaBoost: reweight difficult examples after each weak learner.

Choosing a baseline

For tabular supervised learning, start with a regularized linear model and a random forest or gradient boosting model. The linear model gives a scale-sensitive, interpretable baseline. The tree ensemble gives a nonlinear, interaction-aware baseline. If both fail, inspect data leakage, target quality, and feature definitions before reaching for more complicated models.

7. Workflow, Validation, and Metrics

Classic ML succeeds or fails on data splitting, preprocessing discipline, metric choice, and leakage control.

Validation contracts

Setting Preferred split Why
Independent rows Random train/validation/test or K-fold CV. Rows are exchangeable enough that random splits estimate future data.
Time series Chronological split or rolling-origin validation. Future information must not leak into training.
Grouped data Group split by user, patient, device, account, or entity. The same entity in train and validation inflates performance.
Imbalanced classification Stratified split and threshold evaluation. Accuracy can hide poor minority-class performance.

Metric selection

  • Accuracy: useful for balanced classification with equal error costs.
  • Precision and recall: use when false positives and false negatives have different costs.
  • ROC AUC: threshold-free ranking metric, often optimistic under severe imbalance.
  • PR AUC: better when positives are rare and positive-class quality matters.
  • RMSE: penalizes large regression errors heavily.
  • MAE: robust and easy to explain for regression.
  • Log loss: rewards calibrated probabilities, not just correct labels.

8. Debugging Classic ML

Most failures come from data leakage, bad splits, mismatched preprocessing, target bugs, or objective-metric mismatch.

Symptom Likely cause Check
Validation score is unrealistically high Leakage, duplicated entities, target-derived feature. Audit feature timestamps and split by entity or time.
Model works in notebook but fails in production Preprocessing not packaged with model. Use a pipeline so imputation, scaling, encoding, and estimator are fit together.
SVM or kNN performs poorly Feature scale dominates distance or margin. Standardize numeric features and encode categoricals carefully.
Random forest overfits Trees too deep, leaves too small, leakage, weak validation. Increase min_samples_leaf, cap depth, inspect split design.
K-means clusters are unstable Wrong k, anisotropic data, poor scaling, bad initialization. Run multiple seeds, scale features, compare GMM, DBSCAN, or PCA view. 
# scikit-learn style safety pattern
pipe = Pipeline([
    ("preprocess", ColumnTransformer([
        ("num", StandardScaler(), numeric_columns),
        ("cat", OneHotEncoder(handle_unknown="ignore"), categorical_columns),
    ])),
    ("model", RandomForestClassifier(
        n_estimators=300,
        min_samples_leaf=5,
        random_state=42,
    )),
])

scores = cross_val_score(pipe, X, y, cv=GroupKFold(5), groups=account_id)
