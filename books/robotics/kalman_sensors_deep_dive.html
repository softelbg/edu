Kalman Sensor Filters

A Kalman filter is a recursive estimator that carries both a best estimate and an uncertainty estimate. In robotics, that pair is essential: the robot needs to know not just where it thinks it is, but how much it should trust that belief.

1. State-Space Model

The linear Kalman filter assumes a state vector, a dynamics model, a measurement model, and Gaussian noise. The model can be wrong, but the vocabulary is still useful for designing sensor fusion.

xt=Fxt-1+But+wtzt=Hxt+vt

State

Position, velocity, orientation, bias, object pose, or any quantity that evolves over time.

Process noise Q

Unmodeled acceleration, wheel slip, encoder quantization, unknown contact, or imperfect commands.

Measurement noise R

Camera pixel noise, depth noise, IMU noise, marker jitter, delayed observations, or imperfect detections.

2. Prediction Step

Mean prediction

x-=Fx+Bu

The filter projects the state forward using the motion model before looking at the new sensor measurement.

Covariance prediction

P-=FPFT+Q

Uncertainty grows when the robot moves without observing. Larger Q means the model admits more unmodeled disturbance.

3. Correction Step

Innovation

The innovation is the mismatch between measurement and predicted measurement:

y=z-Hx-

A large innovation can mean real motion, sensor error, data association failure, or a bad model.

Kalman gain

K=P-HT(HP-HT+R)-1

High predicted uncertainty and low measurement noise produce a larger gain. Low predicted uncertainty or noisy measurement produces a smaller gain.

4. Robotics Sensor Fusion Patterns

Arm tracking

Fuse joint encoder estimates with visual end-effector observations. Encoders are fast but model-dependent; vision is slower but catches accumulated geometric error.

Mobile robot

Fuse wheel odometry, IMU acceleration/yaw, visual odometry, and occasional absolute position cues. The filter should account for rate, delay, and coordinate frames.

Bias estimation

Add IMU bias or camera offset to the state vector. Estimating bias prevents systematic drift from being mistaken for real motion.

EKF and UKF

Use EKF for nonlinear models with Jacobians. Use UKF when propagating sigma points is more stable or easier than deriving a Jacobian.

5. Interactive Simulators

Kalman Gain Intuition

Adjust prior and sensor uncertainty. The posterior moves toward the measurement depending on the gain.

Constant-Velocity Tracker

Watch the filter trade off command prediction and noisy position observations.
