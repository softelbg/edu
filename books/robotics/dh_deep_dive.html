Denavit-Hartenberg Kinematics

DH notation is the bridge from a sketch of a serial mechanism to a reproducible chain of homogeneous transforms. The value is not memorizing a table; it is learning to define frames so every joint, link, offset, and tool pose has an auditable coordinate meaning.

1. Why DH Exists

The kinematic bookkeeping problem

A serial manipulator can have many coordinate frames: base, shoulder, elbow, wrist, camera, gripper, and tool-center point. Without a convention, signs and offsets become private assumptions hidden in code. DH gives a compact four-parameter transform for each adjacent pair of frames.

The parameters encode a screw-like move: rotate around an axis, translate along it, shift to the next axis, and twist into alignment.

What DH does not solve

DH does not choose good trajectories, avoid collisions, model elasticity, identify backlash, or calibrate servo zero offsets automatically. It gives nominal rigid-body geometry. Control, calibration, sensing, and safety sit on top.

For low-cost arms, treat the first DH model as a hypothesis. Validate it against measured tool points and visual observations.

2. Frame Assignment

The common source of DH mistakes is not matrix multiplication; it is frame assignment. Draw joint axes first, then assign frames. The table follows from the drawing.

Joint axes

For standard DH, zi-1 is the axis of joint i. Revolute joints rotate around this axis; prismatic joints translate along it.

Common normal

xi points along the common normal from zi-1 to zi. If axes intersect, the normal length is zero; if axes are parallel, choose a consistent normal.

Right-handed frame

yi completes a right-handed coordinate frame. A sign error here flips motion and can look like a wrong inverse-kinematics solution.

3. Standard DH Transform

The standard transform from frame i-1 to frame i is often written as Rz(θi)Tz(di)Tx(ai)Rx(αi).

Ti-1i=[cθi-sθicαisθisαiaicθisθicθicαi-cθisαiaisθi0sαicαidi0001]

Parameter meanings

  • theta: rotation around previous z.
  • d: translation along previous z.
  • a: translation along current x.
  • alpha: rotation around current x.

Composition

The full forward kinematics are the ordered product:

T0n=T01T12⋯Tn-1n

4. Standard vs Modified DH

Modified DH moves the transform components to a different neighboring-frame relationship. It can be more convenient for some robot descriptions, but mixing conventions is a common source of silent errors.

Standard DH order

T_standard = Rz(theta) @ Tz(d) @ Tx(a) @ Rx(alpha)

The parameters describe the relationship from frame i-1 to frame i.

Modified DH order

T_modified = Rx(alpha) @ Tx(a) @ Rz(theta) @ Tz(d)

The parameters are attached differently. Reusing a standard table with modified multiplication gives a wrong robot.

5. Jacobian, Velocity, and Singularities

Forward kinematics gives pose. The Jacobian gives local motion: how small joint changes move the tool. Singularities happen when the Jacobian loses rank, so some Cartesian motion directions become impossible or require extreme joint velocity.

x.=J(q)q.

Interpretation

Columns of J are the tool velocity caused by unit velocity of each joint. For a revolute joint, the translational component is an axis cross product with the tool offset.

Manipulability

A common scalar diagnostic is det(JJT). Near zero means the arm is close to singular.
