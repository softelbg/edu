SLAM: Building a Map While Getting Lost

SLAM is the coupled estimation problem where the robot's pose depends on the map, and the map depends on the robot's pose. Good SLAM systems manage this circularity with probabilistic state estimation, scan alignment, factor graphs, and loop closures.

1. The Core Problem

Localization if map known

Given a map, estimate robot pose from odometry and sensor measurements. This is already uncertain because wheel slip, visual ambiguity, and sensor noise accumulate.

Mapping if pose known

Given exact poses, insert range or visual observations into a map. This is easier, but real poses are not exact. SLAM solves both simultaneously.

The hard part is not drawing obstacles. The hard part is correcting old poses and old map cells when the robot later realizes it has returned to a known place.

2. Occupancy Grid Mapping

An occupancy grid stores the probability that each cell is blocked. Range beams update free space along the ray and occupied evidence near the hit point.

Log-odds form

Log odds makes repeated updates additive:

L_t(m_i) = L_{t-1}(m_i) + inverse_sensor_model(m_i, x_t, z_t) - L_0

Positive values mean occupied; negative values mean free; near zero means unknown.

Failure modes

  • Dynamic objects become false walls.
  • Bad pose estimates smear obstacles.
  • Thin objects disappear at coarse resolution.
  • Specular or transparent objects confuse range sensors.

3. Scan Matching and Visual Alignment

Scan matching estimates the transform that best aligns a new observation with a map or previous scan. In 2D LiDAR this often means matching point sets or occupancy likelihood. In visual SLAM it may mean feature correspondences, direct photometric error, or learned visual descriptors.

ICP

Iterative Closest Point alternates between correspondences and transform fitting. It is intuitive but sensitive to initialization and outliers.

Correlative matching

Search over candidate poses and score how well scan endpoints land on occupied map cells. This is common in grid-based 2D SLAM.

Visual odometry

Track visual features or image patches between frames. It estimates relative camera motion but drifts without global correction.

4. Pose Graph SLAM

A pose graph contains nodes for robot poses and edges for constraints: odometry edges, scan-matching edges, landmark observations, and loop closures. Optimization changes all poses to minimize total error.

Factor graph view

minimize sum_i || e_i(x_a, x_b, z_i) ||^2_{Omega_i}

The information matrix Omega says how much to trust each constraint.

Loop closure

Loop closure detects that the robot has returned to a known place. A correct loop closure can remove long-term drift; a false loop closure can destroy the map.

5. Interactive Simulators

Occupancy Ray Update

Move the robot and beam. Free cells along the ray become lighter; hit cells become darker.

Pose Graph Loop Closure

Odometry drift bends the trajectory. Increase loop-closure strength to see the graph pull the end of the path back toward the start.
