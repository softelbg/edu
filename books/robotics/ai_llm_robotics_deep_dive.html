AI, LLMs, and Agentic Robotics

LLMs and VLMs are powerful interfaces to physical work, but robotics punishes vague text. The useful pattern is layered: language for intent, vision-language for grounding, VLA or skill policies for action proposals, and classical controllers for bounded execution.

1. What AI Models Should Do in Robots

Understand intent

Turn "put the cup near the sink" into objects, constraints, sequence, and success criteria. Ask clarifying questions when the goal is ambiguous.

Ground perception

Map language to visible objects, regions, affordances, hazards, and uncertainties. A VLM can say "the red cup", but the controller needs pose, size, and reachable approach directions.

Choose skills

Select bounded skills such as navigate, inspect, grasp, place, wipe, open, pour, or hand over. Each skill should have typed arguments and safety limits.

A model can be intelligent and still unsafe if it bypasses state estimation, collision checking, force limits, and stop conditions.

2. Vision-Language-Action Models

VLA models connect visual observations and language instructions to robot actions. Some output action tokens, some continuous action chunks, and some high-level commands. The central question is the action representation: what exactly does the model control?

Typical VLA pipeline

camera images + instruction
-> vision encoder
-> language/action transformer
-> action decoder
-> robot command chunk

Action chunks can reduce latency by predicting several future control steps, but the robot still needs feedback and interruption.

Generalization trap

VLA models may understand language broadly but fail on embodiment details: gripper geometry, camera placement, friction, calibration, joint limits, and timing. That is why evaluation must be robot-specific.

RT-2

Google DeepMind presents RT-2 as a VLA model that transfers web-scale vision-language knowledge into robotic control.

OpenVLA

OpenVLA is an open-source 7B VLA trained with robot demonstration data and designed to be fine-tuned for visuomotor control.

GR00T / Gemini Robotics

Current foundation-model robotics work focuses on general-purpose humanoid and dexterous robot behavior, often combining vision, language, action, simulation, and real robot data.

3. Agentic Robot Architecture

Planner

Produces a task graph: inspect scene, identify target, choose grasp affordance, move to pre-grasp, close gripper, lift, verify, recover if needed.

Executor

Owns timing, constraints, and cancellation. It rejects invalid tool calls and reports structured failures to the planner.

Verifier

Checks whether the physical world changed as expected. Vision-only success is weak; combine object state, gripper force, pose, and user confirmation when needed.

Memory

Stores scene facts, calibration, preferred tools, known failure modes, and user-specific constraints. Memory must be inspectable and correctable.

4. Safety Contract

Safe robot AI is not "a better prompt". It is a boundary between high-level reasoning and low-level authority.

Typed skill call

{
  "skill": "pick",
  "object_id": "cup_3",
  "approach": "side",
  "workspace": "table_zone_A",
  "max_speed_m_s": 0.05,
  "force_limit": "low",
  "abort_if": ["human_close", "object_lost", "joint_limit"]
}

Evaluation metrics

  • Task success and recovery success.
  • Unsafe near misses and emergency stops.
  • Object grounding precision.
  • Tool-call schema validity.
  • Human intervention rate.
  • Explanation faithfulness.
